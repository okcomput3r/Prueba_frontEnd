# Prueba Frontend

Prueba de concepto de applicación web tipo dashboard para la empresa Internxt. La aplicación ha sido desarrollada con React + Typescript, y se ha hecho uso de Vite como Build System, y Vercel como host para el repositorio.

## Instrucciones de uso

El proyecto se puede ejecutar tanto en local, ejecutando el comando ```npm run dev```, como a través de la web [accediendo desde este link](https://pruebafrontend-chi.vercel.app/)

## Decisiones en el dessarrollo

Decidí utilizar vite como el Build System del proyecto porque era la opción con la que estoy más familiarizado, y además tiene la opción de hot reload, que es muy util a la hora del  desarrollo, ya que permite iterar rapidamente y agiliza la implementación de nuevos elementos. 

De la misma forma, solo he trabajado con Vercel para el despliegue de aplicaciones interactivas, así que he decidido utilizarlo ya que estaba un poco familiarizado con su funcionamiento, de todas formas es tan sencillo de usar que no haría falta ni experiencia previa.

En cuanto al lenguaje utilizado, en general prefiero utilizar typescript para este tipo de proyectos, ya que me parece que el hecho de tener que especificar los tipos de datos ayuda mucho a aportar claridad al proyecto. Nunca he sido muy fan de los lenguajes debilmente tipados, y esta solución parece remediar un poco ese problema. 

En cuanto al framework, he trabajado con React ya que es con el que estoy más familiarizado, y además si creas un proyecto con vite, se autogenera una plantilla CSS, la cual ha servido de mucha ayuda a la hora de agilizar el desarrollo para este pequeño proyecto. 

## Mejoras

Si ubiese tenido más tiempo, creo que me habría gustado mejorar un poco la estética general de la aplicación, siento que se me ha quedado un poco apagada, y al haber utilizado unicamente logos UNICODE en lugar de imágenes la apariencia de los archivos ha quedado bastante cutre.

También me habría gustado haber hecho algunas de las funciones opcionales, como la de Drag&Drop o el preview, que se que le habría quedado muy bien al producto final, pero se escapan del conocimiento que tengo actualmente de desarrollo web. Quizás con un poco de tiempo invertido en investigar podría haberlo conseguido.
